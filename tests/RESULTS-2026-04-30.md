## Bugs trouvés

### 🔴 BUG-006 — Génération de fichiers absente du déploiement

- **Section** : 4 (Génération fichiers, tous tests G01-G20)
- **Steps** : Demander à l'Assistant comptable "Genere-moi un fichier Excel..." OU "Utilise la balise [FILE:nom.ext]..."
- **Attendu** : Chip téléchargeable apparaît, fichier servi par `/api/files/UUID`
- **Observed** :
  - L'agent répond *"Je ne peux pas générer directement des fichiers Excel"*
  - Aucun pre_prompt en DB n'a la doc `[FILE:...]` (5 agents builtin + 1 Concierge → tous en NO)
- **Cause racine** : Le code `file-generators.ts`, `file-storage.ts`, `chat-stream-files.ts`, `/api/files/[id]/route.ts`, et la modif de `/api/chat/route.ts` (TransformStream + FileDetector) ne sont **pas présents** sur le serveur. Le SQL `add-file-generation-doc.sql` n'a pas été appliqué non plus.
- **Réparation** : ré-déployer la feature complète depuis les sources locales `D:\IA_TPE_PME_POWER\services\app\src\lib\file-*.ts` + `chat-stream-files.ts` + chat route, npm install des deps (`docx`, `exceljs`, `pdfkit`, `marked`), et appliquer le SQL d'enrichissement.

### 🔴 BUG-008 — Pas de chip download même quand l'agent utilise [FILE:...]

- **Section** : 4 (G01-G20)
- **Steps** : Forcer l'agent à émettre `[FILE:charges-2024.xlsx]…[/FILE]` en lui donnant la syntaxe directement

### 🔴 BUG-013 — Réponses tronquées par limite max_tokens (qwen3:14b)

- **Section** : Chat / Génération
- **Steps** : Demander à l'Assistant comptable un Excel avec calculs + balise [FILE:...]
- **Attendu** : Réponse complète avec toutes les lignes du tableau
- **Observed** : Réponse coupée à la 1ère ligne `Loyer 1500 300 1800`, manque Salaires/Électricité/Internet/Fournitures
- **Cause probable** : qwen3:14b est un modèle reasoning. Il consomme ~1500 tokens en `<think>` avant de produire 200 tokens utiles → max_tokens=4096 est dépassé silencieusement
- **Pistes** :
  - Augmenter `max_tokens` à 8192+ pour les agents qwen3:14b
  - Ou activer le filtrage des `<think>` côté Dify (option `model.think.budget`?)
  - Ou repasser sur qwen2.5:14b (non-reasoning) pour le comptable
