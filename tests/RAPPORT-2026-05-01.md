**Conclusions** :
- ✅ **Continuous batching OK** : 8 prompts // → 51s pour le plus lent (vs 8×30s=240s en FIFO pur)
- ✅ **Prefix caching efficace** : qwen3:14b et qwen2.5vl:7b restent chargés 24h grâce à `OLLAMA_KEEP_ALIVE=24h`
- ⚠ **2 réponses dégradées** sur Assistant général (qwen2.5vl:7b) : Q03 (general-simple) et Q07 (general-code) → réponses tronquées. Hypothèse : qwen2.5vl optimisé vision, moins fort en text-only court. Pour ces cas, l'Assistant général devrait peut-être router vers qwen3:14b.
- ⚠ **API blocking + agent-chat (Concierge)** : peut-être nécessite `response_mode: streaming` obligatoire pour agents avec tools. Bug à creuser.
