## TL;DR

**Verdict release client : ❌ NO-GO** en l'état.

L'AI Box a fait des **progrès importants** depuis la v0.1.0 (Marketplaces IA/n8n, MCP, Concierge BoxIA, dossiers de conversations, mémoire long-terme) — mais **plusieurs régressions et bugs critiques** rendent l'expérience cassée pour un client final :

- 🔴 **Le Concierge BoxIA (agent par défaut) ne répond jamais** — bloqué sur "Je structure la réponse..." (React error #418)
- 🔴 **Les agents avancés exposent leur `<think>` au user** (qwen3:14b, chain-of-thought brute en anglais)
- 🔴 **La feature "génération de fichiers" est totalement absente** alors qu'elle était listée comme livrable de la version précédente
- 🔴 **Configurer un agent marketplace activé renvoie "unknown_agent"** — mapping cassé
- 🟠 Plusieurs incohérences de tableau de bord, fuites d'architecture (Dify visible)

**Action immédiate recommandée** : fix BUG-018 (Concierge) + BUG-005 (`<think>`) + BUG-015 (configurer agent marketplace) avant toute démo client.

## Bugs critiques (🔴 P0)

### BUG-018 — Concierge BoxIA bloqué sur "Je structure la réponse..." (jamais affichée)

**Impact** : c'est l'agent par défaut (premier vu par le client). Première impression = écran figé.
**Repro** : `/?agent=concierge` → cliquer "Tout fonctionne bien dans la box ?" → attendre 60s+ → la réponse n'apparaît jamais alors que GPU revient à 0%.
**Cause** : React Error #418 ("hydration error") visible en console (`/_next/static/chunks/4bd1b696-...`). Probablement un setState pendant render dans le composant qui affiche le placeholder. Le streaming arrive mais le composant ne se rafraîchit pas.
**Fix** : tracer l'erreur en mode dev (non-minifié) → identifier le composant Concierge qui crash. Probable que ce soit lié au filtrage des `<think>` ou au state de la conversation auto-nommée.

### BUG-005 — Tags `<think>...</think>` exposés en clair dans le chat

**Impact** : illisibilité des réponses pour tous les agents qui utilisent qwen3:14b (Comptable, RH, Support). Le user voit 200-500 mots de chain-of-thought en anglais avant la vraie réponse.
**Repro** : envoyer n'importe quelle question à l'Assistant comptable.
**Fix** : `MessageMarkdown.tsx` doit pré-traiter le content pour extraire et masquer/replier `<think>...</think>`. Proposer un toggle "Voir le raisonnement" pour ceux qui veulent débugger.
**Note** : Le Concierge BoxIA semble avoir un filtre custom ("Je structure la réponse...") mais qui dysfonctionne (cf BUG-018).

### BUG-006 / BUG-008 / BUG-010 — Génération de fichiers totalement absente

**Impact** : la fonctionnalité phare "L'IA peut produire des Word/Excel/PDF/scripts téléchargeables" n'existe plus côté code déployé.
**Constat** :
- Aucun pre_prompt en DB n'a la doc `[FILE:...]` (6 agents → tous NO)
- Code absent : `lib/file-generators.ts`, `lib/file-storage.ts`, `lib/chat-stream-files.ts`, `/api/files/[id]/route.ts`
- `/api/chat/route.ts` ne contient pas de `FileDetector` (passthrough simple Dify)
- Dossier `/srv/ai-stack/data/generated/` n'existe pas
**Symptôme client** : l'agent répond *"Je ne peux pas générer directement des fichiers Excel"*.
**Fix** : ré-déployer les sources locales (`D:\IA_TPE_PME_POWER\services\app\src\lib\file-*.ts`, etc.) + npm install (`docx`, `exceljs`, `pdfkit`, `marked`) + appliquer `tools/add-file-generation-doc.sql`.

### BUG-013 — Réponses tronquées par max_tokens (qwen3:14b)

**Impact** : sur calculs comptables longs, la réponse est coupée au milieu (souvent dans la 1ère ligne d'un tableau).
**Cause** : qwen3:14b consomme ~1500-2000 tokens en `<think>` avant de produire la vraie réponse. Avec max_tokens=4096 par défaut, le texte utile est rogné.
**Fix** : passer max_tokens à 8192 sur les agents qwen3:14b, OU activer le réglage Dify "exclude thinking from token count" si disponible, OU revenir à qwen2.5:14b (non-reasoning).

### BUG-015 — "Configurer" sur agent marketplace activé → erreur "unknown_agent"

**Impact** : on peut activer un agent marketplace mais on ne peut pas le configurer (pre-prompt, modèle, max_tokens). Les agents marketplace sont donc figés sur leurs valeurs par défaut.
**Repro** : Marketplace IA → activer "Assistant juridique CGV/RGPD" → /agents → cliquer "Configurer" → bandeau rouge "unknown_agent".
**Cause** : `lib/agents.ts` (registre statique builtin) ne contient pas l'agent activé. La route `/api/agents/[slug]` n'a pas de fall-back vers une persistance des agents marketplace.
**Fix** : étendre `resolveAgentMeta()` pour interroger une table/JSON des agents marketplace activés (similaire à ce qui existait dans `custom-agents.json`).

### BUG-020 — Code blocks : pas de bouton "Save as .ext" (sauf HTML qui a "Voir")

**Impact** : pour les langages non-rendables (PowerShell, Bash, Python, JSON, YAML), pas de bouton de téléchargement direct.
**Repro** : demander un script PowerShell → seul "Copier" est dispo.
**Fix** : ré-ajouter le composant `SAVE_EXT_BY_LANG` dans `MessageMarkdown.tsx` (mes commits précédents en avaient une implémentation propre).

## Bugs importants (🟠 P1)

### BUG-002 — Connecteur Pennylane et Import FEC en "fetch failed"

**Localisation** : `/system` (État serveur).
**Impact** : 2 services rouges sur le tableau de bord, alarmants pour le client.
**Investigation** : conteneurs présents ? Healthcheck endpoints implémentés ? À auditer.

### BUG-007 — "Assistant Q&R documents" en DB mais invisible dans /agents

**Repro** : SQL `SELECT name FROM apps` → 6 lignes ; UI → 5 cards.
**Cause** : agent vestigial non mappé dans `lib/agents.ts`.
**Fix** : soit ajouter au registre, soit supprimer l'app Dify orpheline.

### BUG-016 — Suggestions agent marketplace génériques

**Impact** : l'Assistant juridique CGV/RGPD activé propose "Aide-moi à rédiger un email professionnel" / "Quelle est la procédure de demande de congés" — questions hors-sujet pour un agent juridique.
**Cause** : le template marketplace n'injecte pas les `suggested_questions` métier dans la config Dify.
**Fix** : ajouter `suggested_questions` aux templates BoxIA-FR.

### BUG-017 — Click "Configurer" MCP ouvre Dify en nouvel onglet

**Impact** : fuite d'architecture (le client ne devrait jamais voir Dify). Casse la promesse "AI Box = expérience unifiée".
**Repro** : `/integrations/mcp` → "Configurer" sur n'importe quel MCP → ouvre `http://192.168.15.210:8081/tools?category=mcp` (Dify).
**Fix** : implémenter une vraie modale de config MCP côté BoxIA, ou si délégation à Dify gardée, masquer l'URL via reverse-proxy.

### BUG-019 — Tous les agents ont les mêmes `suggested_questions` génériques

**Repro** : ouvrir Assistant général puis Assistant juridique → mêmes 4 questions ("Aide-moi à rédiger...", "Procédure de congés", etc.).
**Fix** : enrichir les pre-prompts ou la table Dify avec des suggestions métier par agent.

### Régression structurelle — Wizard de création d'assistant custom retiré

**Constat** : le bouton "+ Nouvel assistant" et toute la flow `/api/agents/draft-prompt` + `/api/agents/create` sont retirés.
**Bénéfice de la régression** : le Marketplace IA est plus simple pour le client final (clic 1-fois).
**Limite** : impossible de créer un assistant 100% custom sur-mesure (rôle, ton, mots-clés métier) sans passer par Dify directement.
**Recommandation** : garder le marketplace en flow principal MAIS ré-ajouter un bouton "+ Custom" pour l'admin avancé.

### BUG-014 — Banner "Tous configurés sur Qwen2.5-7B local" alors que les agents utilisent qwen3:14b

**Localisation** : Marketplace IA, banner d'info.
**Fix** : MAJ texte ou rendre dynamique en lisant le model effectif.

### Chat — pas d'auto-scroll fluide pendant streaming

**Constat** : pendant le streaming d'une longue réponse, le viewport ne suit pas le contenu nouveau, l'utilisateur doit scroller manuellement.
**Repro** : envoyer un prompt long à l'Assistant comptable.
**Fix** : ajouter `scrollIntoView` sur le dernier chunk reçu.

### Onglet Discuter — défilement horizontal des tags de conversation

**Constat** : la barre de tags `#urgent #client-acme #compta-2026 #ms365` est tronquée à droite, scrollbar horizontale visible — mauvaise UX.
**Fix** : repli en grid 2 lignes, ou collapse "Plus de tags…".

## Bugs cosmétiques (🟡 P2)

### BUG-003 — Status agrégat incohérent (/system)

Banner vert "Tous les services sont opérationnels" alors que 2 services en `fetch failed`. Logique d'agrégat à corriger.

### BUG-004 — Branche git affichée comme "—" / version "unknown" (/settings)

`git rev-parse` non disponible dans le container ou `.git` non monté. Fix : `ARG GIT_BRANCH` au build.

### Page-level cosmetic — grid 2 colonnes vs 3

Les pages Marketplace IA / n8n / MCP affichent 3 colonnes de cards mais sur viewport 1568px il y a beaucoup d'espace vide à droite des cards. Ajuster `lg:grid-cols-3` à 4 colonnes pour les écrans larges, ou agrandir les cards.

### Header global — tags conversations doivent dépendre de l'agent courant

Les tags `#urgent #client-acme #compta-2026 #ms365` sont affichés sur tous les agents, même ceux qui n'ont aucune conv taggée — bruyant.

## Conclusion

L'AI Box montre une **direction produit prometteuse** : marketplaces unifiées, MCP, mémoire, dossiers, tags. Ce sont les ingrédients d'un produit différencié sur le marché TPE/PME.

Mais **deux trous structurels rendent la box non-livrable en l'état** :
- L'agent par défaut Concierge ne répond jamais (BUG-018)
- Les agents reasoning exposent leur `<think>` (BUG-005) — y compris sur les agents métier (Comptable, RH)

Une fois ces 2 bugs P0 fixés, la box sera **utilisable en démo**. Pour une **release client commerciale**, il faut aussi régler BUG-015, BUG-013, et réintégrer la génération de fichiers.

**Estimé temps de correction P0 + P1 critiques** : 2-3 jours de dev focus.
